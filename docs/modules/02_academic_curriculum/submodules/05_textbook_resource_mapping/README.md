Map textbooks to curriculum topics, maintain comprehensive resource library, track digital resources, ensure resource availability, manage resource distribution, monitor resource usage, and provide alternative learning materials.

- **Digital Resources**: Videos, simulations, interactive tools
- **Supplementary Materials**: Workbooks, practice sheets
- **Resource Library**: Centralized resource repository

## üìä DATABASE SCHEMA

### Resource Mapping Table

```sql
CREATE TABLE resource_mapping (
  resource_id INT PRIMARY KEY AUTO_INCREMENT,
  
  -- Resource Details
  resource_type ENUM(
    'TEXTBOOK', 'REFERENCE_BOOK', 'WORKBOOK',
    'VIDEO', 'SIMULATION', 'INTERACTIVE',
    'WORKSHEET', 'ASSESSMENT', 'GAME'
  ) NOT NULL,
  
  title VARCHAR(500) NOT NULL,
  author VARCHAR(300),
  publisher VARCHAR(200),
  
  -- Book Details
  isbn VARCHAR(20),
  edition VARCHAR(50),
  publication_year INT,
  price DECIMAL(10,2),
  pages INT,
  
  -- Digital Resource Details
  url VARCHAR(1000),
  duration_minutes INT,
  language VARCHAR(100),
  access_type ENUM('FREE', 'SUBSCRIPTION', 'PAID') DEFAULT 'FREE',
  
  -- Curriculum Mapping
  subject_id INT NOT NULL,
  grade VARCHAR(10) NOT NULL,
  chapter_id INT,
  topic VARCHAR(300),
  
  -- Usage
  is_prescribed BOOLEAN DEFAULT FALSE,
  is_recommended BOOLEAN DEFAULT TRUE,
  
  -- Ratings
  rating DECIMAL(3,2),
  reviews_count INT DEFAULT 0,
  
  -- Metadata
  created_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  modified_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  
  -- Foreign Keys
  FOREIGN KEY (subject_id) REFERENCES subjects(subject_id),
  
  -- Indexes
  INDEX idx_subject_grade (subject_id, grade),
  INDEX idx_resource_type (resource_type),
  INDEX idx_prescribed (is_prescribed)
);
```

### Resource Distribution Table

```sql
CREATE TABLE resource_distribution (
  distribution_id INT PRIMARY KEY AUTO_INCREMENT,
  
  -- Student & Resource
  student_id INT NOT NULL,
  resource_id INT NOT NULL,
  
  -- Distribution Details
  distribution_date DATE NOT NULL,
  expected_return_date DATE,
  actual_return_date DATE,
  
  -- Condition
  condition_at_issue ENUM('NEW', 'EXCELLENT', 'GOOD', 'FAIR') NOT NULL,
  condition_at_return ENUM('EXCELLENT', 'GOOD', 'FAIR', 'POOR', 'LOST'),
  
  -- Fines
  damage_fine DECIMAL(10,2) DEFAULT 0,
  late_fine DECIMAL(10,2) DEFAULT 0,
  total_fine DECIMAL(10,2) DEFAULT 0,
  fine_paid BOOLEAN DEFAULT FALSE,
  
  -- Status
  distribution_status ENUM('ISSUED', 'RETURNED', 'LOST', 'DAMAGED') DEFAULT 'ISSUED',
  
  -- Metadata
  issued_by INT,
  returned_to INT,
  created_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  
  -- Foreign Keys
  FOREIGN KEY (student_id) REFERENCES students(student_id) ON DELETE CASCADE,
  FOREIGN KEY (resource_id) REFERENCES resource_mapping(resource_id),
  
  -- Indexes
  INDEX idx_student_id (student_id),
  INDEX idx_status (distribution_status),
  INDEX idx_dates (distribution_date, expected_return_date)
);
```

### Rule 1: Resource Availability Check

```javascript
FUNCTION check_resource_availability(resource_id, required_quantity) {
  resource = GET_RESOURCE(resource_id)
  
  // Get total inventory
  total_inventory = QUERY("
    SELECT quantity
    FROM resource_inventory
    WHERE resource_id = ?
  ", [resource_id])
  
  // Get currently issued
  issued_count = QUERY("
    SELECT COUNT(*) as count
    FROM resource_distribution
    WHERE resource_id = ?
    AND distribution_status = 'ISSUED'
  ", [resource_id])
  
  available = total_inventory.quantity - issued_count.count
  
  IF available < required_quantity {
    RETURN {
      available: FALSE,
      message: `Only ${available} copies available, ${required_quantity} required`,
      shortage: required_quantity - available
    }
  }
  
  RETURN {
    available: TRUE,
    available_quantity: available
  }
}
```
