## üìã OVERVIEW

The Special Needs & Accommodations submodule manages comprehensive support for students with learning differences, disabilities, and special educational needs. It tracks Individual Education Plans (IEPs), classroom accommodations, assistive technologies, therapy sessions, progress monitoring, and ensures inclusive education compliance with RPWD Act 2016 (Rights of Persons with Disabilities Act).

### Purpose

Identify students with special needs, create and maintain IEPs, provide appropriate accommodations, track therapy and intervention progress, ensure accessibility, coordinate with special educators and therapists, maintain confidentiality, and ensure compliance with disability rights legislation.

### Scope

- **IEP Management**: Individual Education Plans
- **Accommodations**: Classroom and exam modifications
- **Therapies**: Speech, occupational, behavioral
- **Assistive Technology**: Tools and devices
- **Progress Tracking**: Regular assessments
- **Parent Collaboration**: Regular meetings
- **Legal Compliance**: RPWD Act 2016, RTE Act

### Special Needs Profile Table

```sql
CREATE TABLE student_special_needs (
  special_needs_id INT PRIMARY KEY AUTO_INCREMENT,
  student_id INT NOT NULL,
  
  -- Diagnosis
  diagnosis VARCHAR(200) NOT NULL,
  diagnosis_category ENUM(
    'LEARNING_DISABILITY', 'ADHD', 'AUTISM', 'PHYSICAL_DISABILITY',
    'VISUAL_IMPAIRMENT', 'HEARING_IMPAIRMENT', 'SPEECH_DISORDER', 'OTHER'
  ) NOT NULL,
  
  severity ENUM('MILD', 'MODERATE', 'SEVERE') NOT NULL,
  
  -- Diagnosis Details
  diagnosed_by VARCHAR(200),
  diagnosis_date DATE,
  assessment_report_path VARCHAR(500),
  
  -- IEP
  has_iep BOOLEAN DEFAULT TRUE,
  iep_start_date DATE,
  iep_end_date DATE,
  iep_document_path VARCHAR(500),
  
  -- Services
  special_education BOOLEAN DEFAULT FALSE,
  speech_therapy BOOLEAN DEFAULT FALSE,
  occupational_therapy BOOLEAN DEFAULT FALSE,
  behavioral_therapy BOOLEAN DEFAULT FALSE,
  counseling BOOLEAN DEFAULT FALSE,
  
  -- Status
  is_active BOOLEAN DEFAULT TRUE,
  
  -- Confidentiality
  confidential BOOLEAN DEFAULT TRUE,
  parent_consent BOOLEAN DEFAULT FALSE,
  
  -- Metadata
  created_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  modified_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  notes TEXT,
  
  -- Foreign Keys
  FOREIGN KEY (student_id) REFERENCES students(student_id) ON DELETE CASCADE,
  
  -- Indexes
  INDEX idx_student_id (student_id),
  INDEX idx_category (diagnosis_category),
  INDEX idx_active (is_active)
);
```

### Rule 1: IEP Review Reminder

```javascript
FUNCTION check_iep_review_due() {
  // Run monthly
  today = CURRENT_DATE()
  
  // Find IEPs due for review in next 30 days
  due_ieps = QUERY("
    SELECT sn.*, s.full_name, s.admission_number,
           f.father_email, f.father_mobile
    FROM student_special_needs sn
    JOIN students s ON sn.student_id = s.student_id
    JOIN families f ON s.family_id = f.family_id
    WHERE sn.is_active = TRUE
    AND sn.has_iep = TRUE
    AND sn.iep_end_date BETWEEN ? AND DATE_ADD(?, INTERVAL 30 DAY)
  ", [today, today])
  
  FOR each iep IN due_ieps {
    // Send reminder to IEP team
    SEND_EMAIL(iep.father_email, {
      subject: "IEP Review Due",
      message: `IEP review for ${iep.full_name} due on ${iep.iep_end_date}`
    })
    
    // Notify special educator
    SEND_NOTIFICATION("special_educator", {
      title: "IEP Review Due",
      message: `${iep.full_name} IEP review due in ${DAYS_UNTIL(iep.iep_end_date)} days`
    })
  }
}
```
