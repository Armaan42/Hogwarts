## 📋 OVERVIEW

The Internship Management submodule provides a complete platform for managing student internships — from opportunity discovery and application through placement, monitoring, evaluation, and academic credit mapping. It connects students with industry partners, tracks internship progress, facilitates mentor feedback, and ensures compliance with academic requirements for internship-based credits in professional programs.

### Purpose

Bridge the gap between academic learning and industry experience by providing students (Grade 11-12, undergraduate, and postgraduate) with structured internship opportunities, tracking their professional development, ensuring quality industry mentorship, and converting internship outcomes into academic credits and employability metrics.

### Scope

- **Opportunity Management**: Industry partner database, internship posting, application tracking
- **Student Matching**: Skill-based matching, preference mapping, interview scheduling
- **Placement Tracking**: Offer management, joining formalities, internship period tracking
- **Monitoring & Evaluation**: Weekly logs, mentor feedback, mid-term reviews
- **Academic Integration**: Credit mapping, grade conversion, project documentation
- **Company Relations**: MOU management, partner ratings, feedback loops
- **Compliance**: Labor law compliance, stipend tracking, insurance coverage

## 📊 DATA FIELDS

### Internship Record

```sql
CREATE TABLE career_internships (
    internship_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    student_id INT NOT NULL,
    company_id INT NOT NULL,
    
    -- Internship Details
    role_title VARCHAR(100),
    department VARCHAR(100),
    location VARCHAR(100),
    work_mode ENUM('ONSITE', 'REMOTE', 'HYBRID'),
    start_date DATE,
    end_date DATE,
    duration_weeks INT,
    
    -- Financial
    stipend_monthly DECIMAL(10,2) DEFAULT 0,
    total_stipend DECIMAL(10,2) DEFAULT 0,
    stipend_status ENUM('PAID', 'PARTIAL', 'PENDING', 'WAIVED'),
    
    -- Academic
    academic_credits DECIMAL(4,1),
    mid_term_score DECIMAL(5,2),
    final_score DECIMAL(5,2),
    grade CHAR(2),
    project_title VARCHAR(200),
    
    -- Evaluation
    mentor_name VARCHAR(100),
    mentor_email VARCHAR(100),
    mentor_rating DECIMAL(3,1),
    
    -- Outcome
    ppo_offered BOOLEAN DEFAULT FALSE,
    ppo_accepted BOOLEAN DEFAULT FALSE,
    ppo_package DECIMAL(10,2),
    
    -- Status
    status ENUM('APPLIED', 'SHORTLISTED', 'INTERVIEW', 'SELECTED', 'OFFER_ACCEPTED', 'ACTIVE', 'COMPLETED', 'TERMINATED', 'WITHDRAWN'),
    
    -- Compliance
    agreement_signed BOOLEAN DEFAULT FALSE,
    insurance_active BOOLEAN DEFAULT FALSE,
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_student (student_id),
    INDEX idx_company (company_id),
    INDEX idx_status (status)
);

CREATE TABLE career_internship_weekly_logs (
    log_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    internship_id BIGINT NOT NULL,
    week_number INT NOT NULL,
    
    -- Log Content
    tasks_completed TEXT,
    skills_developed TEXT,
    challenges TEXT,
    hours_logged DECIMAL(5,1),
    attendance_days INT,
    
    -- Mentor Feedback
    mentor_feedback TEXT,
    mentor_rating DECIMAL(3,1),
    
    submitted_date DATE,
    status ENUM('DRAFT', 'SUBMITTED', 'REVIEWED'),
    
    FOREIGN KEY (internship_id) REFERENCES career_internships(internship_id)
);
```

## 📋 BUSINESS RULES

### Rule 1: Eligibility Validation
```
FUNCTION validate_internship_eligibility(student, internship):
  IF student.cgpa < internship.min_cgpa:
    RETURN {eligible: FALSE, reason: "CGPA {student.cgpa} below minimum {internship.min_cgpa}"}
  END IF
  
  IF student.active_backlogs > 0:
    RETURN {eligible: FALSE, reason: "Cannot apply with active backlogs"}
  END IF
  
  IF student.current_internship IS NOT NULL AND student.current_internship.status == 'ACTIVE':
    RETURN {eligible: FALSE, reason: "Already in an active internship"}
  END IF
  
  IF student.discipline_issues > 0:
    REQUIRE_HOD_APPROVAL: "Student has disciplinary record. HOD clearance needed."
  END IF
  
  RETURN {eligible: TRUE}
END FUNCTION
```

### Rule 2: Weekly Log Compliance
```
FUNCTION check_weekly_log_compliance(internship):
  current_week = CALCULATE_WEEK(internship.start_date, TODAY)
  submitted_logs = COUNT(logs WHERE internship_id = internship.id AND status IN ('SUBMITTED', 'REVIEWED'))
  
  IF current_week - submitted_logs > 2:
    ALERT student: "You have {current_week - submitted_logs} pending weekly logs. Submit by Friday."
    ALERT academic_mentor: "{student.name} has overdue weekly logs."
    
    IF current_week - submitted_logs > 4:
      ESCALATE_TO_HOD: "Chronic non-submission of internship logs"
      FLAG_CREDIT: "Credit at risk due to non-compliance"
    END IF
  END IF
END FUNCTION
```

## ⚠️ EDGE CASES

### Edge Case 1: Company Terminates Intern Early
- **Scenario:** TCS terminates Aarav's internship in Week 10 (of 24) due to company restructuring.
- **Resolution:** Career Services contacts company for formal termination letter. Student's completed weeks (10) evaluated for partial credit (proportional: 10/24 × 6 = 2.5 credits). Student reassigned to another company or given alternative project. Internship status changed to "TERMINATED_BY_COMPANY". Company's reliability rating updated.

### Edge Case 2: Student Wants to Switch Company Mid-Internship
- **Scenario:** Student Priya finds a better opportunity at Google after 4 weeks at Wipro.
- **Resolution:** Not allowed without HOD approval. If approved: 15-day notice required to current company. New internship starts fresh (previous weeks don't count). Original company notified formally. Student's record reflects the switch for future reference.

### Edge Case 3: Unpaid Internship at NGO
- **Scenario:** Student wants to intern at an NGO that offers no stipend.
- **Resolution:** System allows stipend = ₹0 with explicit student consent. Academic credits awarded normally. Insurance still required (institution-funded). Career Services verifies NGO legitimacy before approval.
